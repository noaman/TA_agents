Single Agent

Handles one discrete, well-defined task autonomously. Stateless per invocation. Examples: CV parsing, salary threshold validation, calendar availability check, document verification.

  • ◉ Form Validation Agent
  • ◉ CV/Resume Parsing Agent
  • ◉ Salary Validation Agent
  • ◉ Calendar Availability Agent
  • ◉ Contract Generation Agent
  • ◉ Document Verification Agent
  • ◉ Currency Conversion Agent
  • ◉ Duplicate Detection Agent

Multi-Agent Workflow

Orchestrated chain of agents collaborating on a complex process. Maintains shared state, routes decisions, and handles branching logic across multiple steps and roles.

  • ◉ Approval Chain Orchestrator
  • ◉ Sourcing Pipeline Orchestrator
  • ◉ Interview Scheduling Coordinator
  • ◉ Offer Approval Workflow
  • ◉ Contract Issuance Pipeline
  • ◉ Onboarding Task Orchestrator
  • ◉ Agency Submission Pipeline
  • ◉ Shortlist Review Workflow

Agentic Automation

Event-driven autonomous agents that monitor, escalate, and act without human intervention. Continuously watches system state and triggers actions based on rules and SLA thresholds.

  • ◉ OLA Breach Monitor
  • ◉ Escalation Matrix Engine
  • ◉ Auto-Regret Notification Agent
  • ◉ Feedback Reminder Agent
  • ◉ Day-1 Readiness Checker
  • ◉ Portal Access Lifecycle Agent
  • ◉ Audit Trail Logger
  • ◉ Data Retention Policy Agent
Design Principles: Each agent is scoped to a single responsibility. Multi-agent workflows orchestrate agents via a shared message bus. Agentic automation runs as background daemons, triggered by system events (status changes, timer expirations, threshold breaches). All agents log to a central audit trail. Configuration (SLAs, escalation rules, routing) is externalized to the Admin Center — no code changes required for routine adjustments.
