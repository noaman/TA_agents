End-to-End Recruitment Lifecycle — Agent Overlay

Single Agent

Handles one discrete, well-defined task autonomously. Stateless per invocation. Examples: CV parsing, salary threshold validation, calendar availability check, document verification.

  • ▶ Form Validation Agent
  • ▶ CV/Resume Parsing Agent
  • ▶ Salary Validation Agent
  • ▶ Calendar Availability Agent
  • ▶ Contract Generation Agent
  • ▶ Document Verification Agent
  • ▶ Currency Conversion Agent
  • ▶ Duplicate Detection Agent

Multi-Agent Workflow

Orchestrated chain of agents collaborating on a complex process. Maintains shared state, routes decisions, and handles branching logic across multiple steps and roles.

  • ▶ Approval Chain Orchestrator
  • ▶ Sourcing Pipeline Orchestrator
  • ▶ Interview Scheduling Coordinator
  • ▶ Offer Approval Workflow
  • ▶ Contract Issuance Pipeline
  • ▶ Onboarding Task Orchestrator
  • ▶ Agency Submission Pipeline
  • ▶ Shortlist Review Workflow

Agentic Automation

Event-driven autonomous agents that monitor, escalate, and act without human intervention. Continuously watches system state and triggers actions based on rules and SLA thresholds.

  • ▶ OLA Breach Monitor
  • ▶ Escalation Matrix Engine
  • ▶ Auto-Regret Notification Agent
  • ▶ Feedback Reminder Agent
  • ▶ Day-1 Readiness Checker
  • ▶ Portal Access Lifecycle Agent
  • ▶ Audit Trail Logger
  • ▶ Data Retention Policy Agent
Design Principles: Each agent is scoped to a single responsibility. Multi-agent workflows orchestrate agents via a shared message bus. Agentic automation runs as background daemons, triggered by system events (status changes, timer expirations, threshold breaches). All agents log to a central audit trail. Configuration (SLAs, escalation rules, routing) is externalized to the Admin Center — no code changes required for routine adjustments.

2.2 — Candidate Sourcing & Shortlisting

Multi-channel candidate acquisition, screening, and HM shortlist review with up to 3 iteration cycles.

Sourcing Pipeline — Agent Mapping

Single Agent CV Parsing Agent

  • → Extracts structured data from uploaded CVs/resumes (PDF, DOCX)
  • → Normalizes fields: name, email, phone, experience, education, skills
  • → Tags candidate source automatically (LinkedIn, Referral, Agency, Direct)
  • → Handles multi-language documents (English & Arabic)
Document ParserNLP Engine

Single Agent Duplicate Detection Agent

  • → Fuzzy-matches incoming candidates against existing database
  • → Checks email, phone, name combinations with configurable thresholds
  • → Links candidate to historical applications if match found
  • → Alerts recruiter for manual review on partial matches

Multi-Agent Sourcing Pipeline Orchestrator

  • → Aggregates candidates from all channels into a unified pipeline
  • → Coordinates LinkedIn RSC integration for InMail tracking & profile linking
  • → Routes agency submissions through the parsing, dedup, screening flow
  • → Tracks referral source linkage for employee referral program
  • → Posts jobs to career page and LinkedIn simultaneously
LinkedIn RSC APICareer Page CMSAgency Portal

Multi-Agent Shortlist Review Workflow

  • → Notifies HM when shortlist is ready for review
  • → Tracks Accept/Reject decisions per candidate
  • → Enforces max 3 shortlist iterations rule
  • → On all-reject, routes back to intake meeting after 3 iterations
  • → On accept, enables ranking and moves to interview stage

Automation Auto-Regret Notification Agent

  • → Triggers customized regret emails for rejected candidates
  • → Templates customized by level, company, and country
  • → Scheduled delivery: next business day at 9 AM
  • → Updates candidate portal access status upon final rejection
Email ServiceTemplate EngineScheduler

Automation Source Effectiveness Tracker

  • → Continuously computes conversion rates per candidate source
  • → Feeds data to Power BI dashboards for quarterly analysis
  • → Tracks full funnel: submissions, screened, interviewed, offered, hired

Interview Lifecycle — Agent Mapping

Single Agent Calendar Availability Agent

  • → Queries Outlook/Exchange for interviewer free/busy slots
  • → Generates MS Bookings link for candidate self-scheduling
  • → Creates calendar invite with: round number, date, time, location/link, attendees

2.4 — Offer Request, Approval & Validation

Financial validation, multi-stage approval with salary comparison, and compliance enforcement before offer extension.

Offer Approval Chain — Agent Mapping

Single Agent Salary Validation Agent

  • → Compares expected salary against current salary
  • → Flags when delta exceeds configurable threshold (default: 30%)
  • → Requires explicit override justification before proceeding
  • → Validates salary data is populated for the correct company/country

Single Agent Currency Conversion Agent

  • → Converts all salary figures to SAR for standardized comparison
  • → Handles multi-entity, multi-country salary inputs
  • → Uses configurable exchange rates maintained in Admin Center

Single Agent Comparison View Generator

  • → Auto-generates side-by-side table for CPO review
  • → Displays: current salary, expected salary, OD comments, proposed offer
  • → Highlights variances and OD analysis notes

Single Agent Pre-requisite Checker

  • → Blocks offer issuance until required documents are uploaded
  • → Company and country-specific document requirements
  • → Checks signed application form via Candidate Portal upload

Multi-Agent Offer Approval Orchestrator

  • → Routes: Recruiter, then OD Team (add details), then CPO (final decision)
  • → CPO actions: Approve / Reject (mandatory justification) / Add Comments
  • → Notifies recruiter of final decision immediately
  • → Maintains full approval audit trail with timestamps

Automation Offer Approval OLA Monitor

  • → Tracks time-in-stage for OD and CPO approval steps
  • → Fires escalation alerts per configurable OLA thresholds
  • → Reports OLA compliance rate to leadership dashboards

2.7 — Candidate Portal & Job Posting

Self-service portal for job discovery, application tracking, offer response, onboarding forms, and support — with lifecycle-managed access.

Candidate Portal Architecture — Agent Mapping

Single Agent Document Verification Agent

  • → Validates uploaded documents against country-specific requirements
  • → Checks file format, size, and completeness
  • → Dynamic document list: changes based on nationality (KSA vs. non-KSA)
  • → Tracks status per document: Pending Upload, Uploaded, Verified

Single Agent Support Ticket Agent

  • → Chat-style interface for candidate support requests
  • → Creates structured tickets with: type, subject, description
  • → Routes to appropriate team based on request type
  • → Tracks resolution time for KPI reporting

Automation Status Update Notifier

  • → Pushes real-time status updates to candidate portal dashboard
  • → Sends email notifications on key status changes
  • → Welcoming material and checklist presented upon offer acceptance

Automation Portal Access Lifecycle Agent

  • → Automatically revokes portal access upon final rejection
  • → Revokes after defined inactivity period post-completion
  • → Manages save-draft and form-locking logic
  • → Supports recruiter-initiated form unlock for corrections
Key UX Requirements: Responsive (mobile + desktop), bilingual (English & Arabic), save-draft for lengthy forms, form locking after submission with recruiter unlock capability. Candidate can only view their own data — strict privacy enforcement.

2.8 — External Agency Management

Secure agency portal with strict data isolation, candidate submission, milestone-based invoicing, and performance tracking.

Agency Workflow — Agent Mapping

Single Agent Data Isolation Enforcer

  • → Strictly enforces agency-level data segregation
  • → Each agency sees only their own submitted candidates
  • → Filters all queries by authenticated agency account ID
  • → Prevents cross-agency data leakage at the query layer

Single Agent Invoice Validation Agent

  • → Validates invoice fields: amount, date, comments, PDF attachment
  • → Verifies candidate has reached required milestone status
  • → Checks against agency agreement terms if configured

Multi-Agent Invoice Approval Workflow

  • → Routes: TA Director, then OD Director / Finance Approver
  • → Attaches invoice PDF and candidate reference to each step
  • → On approval, triggers finance notification with full documentation

Automation Agency Performance Monitor

  • → Tracks: candidates submitted vs. interviewed vs. hired per agency
  • → Computes shortlist conversion rate and cost-per-hire
  • → Feeds quarterly agency performance dashboard in Power BI
  • → Surfaces underperforming agencies for TA Director review

Integrations & Tooling Requirements

External systems, APIs, and infrastructure required to power the agentic architecture across all process stages.

Integration Landscape

Integration Microsoft Outlook / Exchange

  • → Purpose: Interview scheduling, intake meeting invites, calendar availability
  • → Protocol: Microsoft Graph API (REST)
  • → Used By: Calendar Availability Agent, Interview Scheduling Coordinator
  • → Data Flow: Bi-directional (send invites, read free/busy)

Integration Microsoft Bookings

  • → Purpose: Self-service interview slot selection by candidates/interviewers
  • → Protocol: Bookings API or embedded link generation
  • → Used By: Calendar Availability Agent
  • → Data Flow: Outbound (generate booking links)

Integration LinkedIn Recruiter System Connect (RSC)

  • → Purpose: InMail tracking, job posting, profile linking, project association
  • → Protocol: LinkedIn RSC API
  • → Used By: Sourcing Pipeline Orchestrator, Source Effectiveness Tracker
  • → Config: Toggle enable/disable via Admin Center (per-project or global)

Integration Power BI / Visualization Tool

  • → Purpose: Dashboards, KPI reporting, executive digests, funnel analysis
  • → Protocol: Direct database connection or scheduled data feed via API
  • → Used By: All monitoring agents, KPI engines, reporting layer
  • → Data: Optimized with Choice fields for efficient BI consumption

Integration Corporate SSO / Identity Provider

  • → Purpose: User authentication, role mapping, access control enforcement
  • → Protocol: SAML 2.0 / OpenID Connect (OIDC)
  • → Used By: All portal access, RBAC enforcement across 14+ roles
  • → Security: Maps corporate directory groups to application roles

Integration ERP System

  • → Purpose: Position status updates (Open/Filled), employee record creation
  • → Protocol: REST API
  • → Used By: Form Validation Agent, Onboarding Orchestrator (BS Team)
  • → Data Flow: Bi-directional (read positions, write new employees)

Integration Digital Signature Service

  • → Purpose: CPO/HR GM contract signing, candidate offer letter signing
  • → Protocol: Digital Signature API (e.g., DocuSign, Adobe Sign)
  • → Used By: Digital Signature Orchestrator, Contract Generation Agent

Integration Email / Notification Service

  • → Purpose: All automated notifications, regret emails, alerts, reminders
  • → Protocol: SMTP / MS Graph Mail API
  • → Used By: All notification agents, escalation engine, OLA monitors
  • → Config: Templates managed via Admin Center (RACI-based routing)

Admin Center

i18n Engine

Bi-directional language support (English and Arabic). All labels, messages, templates, and contract content translatable.
